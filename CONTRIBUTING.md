# Contributing to LocalMind

We appreciate your interest in contributing to LocalMind! Here's how you can help:

## How to Contribute

1. **Fork the repository** on GitHub
2. **Create a feature branch** (`git checkout -b feature/your-feature-name`)
3. **Make your changes** and test them thoroughly
4. **Commit your changes** with clear, descriptive messages
5. **Push to your fork** (`git push origin feature/your-feature-name`)
6. **Open a Pull Request** with a clear description of your changes

## Code Style

- Follow PEP 8 guidelines
- Use meaningful variable and function names
- Add docstrings to functions and modules
- Keep functions focused and single-purpose

## Areas for Contribution

- New integrations (calendar, email, task managers, etc.)
- Additional dashboard panels or visualizations
- Performance improvements
- Documentation and examples
- Bug fixes and issue reports

## Reporting Issues

If you find a bug or have a suggestion, please:

1. Check existing issues first
2. Open a new issue with a clear title and description
3. Include steps to reproduce (for bugs)
4. Provide your environment details (OS, Python version, Ollama model)

## Questions?

Feel free to open an issue or discussion if you have questions about contributing!

Thank you for helping make LocalMind better!
